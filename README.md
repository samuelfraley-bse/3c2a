# Foothill CV Football Scraper

Scrapes 3C2A football data from `3c2asports.org` into per-season CSVs with enriched field position data.

## Pipeline Overview

For each season, run these four steps in order:

```
Step 1 — scrape_season.py            → standings, schedule, games, plays
Step 2 — generate_crosswalk_draft.py → prefix_crosswalk_draft.csv
Step 3 — [Claude] fill crosswalk     → prefix_crosswalk.csv
Step 4 — enrich_plays.py             → plays_with_side.csv
```

---

## Step-by-Step: Adding a New Season

### Step 1 — Scrape the season

```powershell
python scrape_season.py --season 2024-25
```

This hits the 3C2A standings page for that season, follows every team's schedule, and scrapes all play-by-play. Outputs under `outputs/2024-25/`:

- `standings.csv` — one row per team
- `schedule.csv` — one row per team-game
- `games.csv` — one row per unique game
- `plays.csv` — all play-by-play rows

If the scrape is interrupted, rerun with `--plays-only` to resume without re-scraping standings/schedule:

```powershell
python scrape_season.py --season 2024-25 --plays-only
```

To rescrape specific games (e.g. after a parse fix):

```powershell
python scrape_season.py --season 2024-25 --game-ids 20241018_1561 20241025_abcd
```

---

### Step 2 — Generate the crosswalk draft

```powershell
python generate_crosswalk_draft.py --season 2024-25
```

Reads `plays.csv` and `games.csv` to extract the two field position prefixes used in each game. Outputs:

- `outputs/2024-25/prefix_crosswalk_draft.csv`

Columns: `game_id`, `prefix_a`, `prefix_b`, `canonical_a`, `canonical_b` *(blank)*, `team_1`, `team_2`, `note`

---

### Step 3 — Fill the crosswalk with Claude

Open a new Claude conversation and paste this prompt:

> I have a football play-by-play crosswalk CSV. Each row is one game. I need you to fill in `canonical_a` and `canonical_b` — the full team name that each prefix belongs to. You know the two teams in the game from `team_1` and `team_2`. Match each prefix to one of those two teams. Here is the CSV:
>
> [paste the contents of `prefix_crosswalk_draft.csv`]

Claude will return the CSV with `canonical_a` and `canonical_b` filled in. Save that as:

```
outputs/2024-25/prefix_crosswalk.csv
```

Keep the draft file untouched as a reference. The filled `prefix_crosswalk.csv` is what Step 4 reads.

**Tips for the Claude prompt:**
- If there are only a handful of unresolved rows, paste just those rows rather than the full file
- For ambiguous prefixes like `MT. SAN` (could be Mt. San Antonio or Mt. San Jacinto), both teams are in `team_1`/`team_2` so Claude can resolve it unambiguously
- Double-check any rows where `note` contains a warning (more than two prefixes found)

---

### Step 4 — Enrich plays with field position side

```powershell
python enrich_plays.py --season 2024-25
```

Joins `plays.csv` against `prefix_crosswalk.csv` to add three columns:

- `field_pos_side`: `own` (ball in offense's own territory) or `opponent`
- `yardline_raw`: numeric yardline, e.g. `FOOT32` → `32`
- `yardline_100`: yards from opponent end zone, e.g. own 25 → `75`, opponent 25 → `25`

Output: `outputs/2024-25/plays_with_side.csv` — this is the analysis-ready file.

## Output Files

All outputs live under `outputs/{season}/`.

### `standings.csv`
One row per team. Source of canonical team names for the season.

### `schedule.csv`
One row per team-game scraped from a team schedule page. Key columns: `team_name`, `game_id`, `home_away`, `opponent`, `pbp_url`, `schedule_home`, `schedule_away`.

### `games.csv`
One row per unique game slug. Built by grouping schedule rows by `game_id`. Key columns: `game_id`, `pbp_url`, `home_team_canonical`, `away_team_canonical`, `team_1`, `team_2`, `pairing_status`.

`pairing_status` values:
- `paired` — both team rows were scraped (expected)
- `single-sided` — only one team's schedule was in standings
- `duplicate-rows`, `over-paired`, `incomplete` — problematic, investigate

### `plays.csv`
One row per play. Key columns: `game_id`, `quarter`, `down`, `distance`, `field_position`, `offense`, `defense`, `play_type`, `raw_text`, `schedule_home`, `schedule_away`.

### `prefix_crosswalk_draft.csv`
Generated by Step 2. Per-game prefix pairs with blank canonical columns — input for Claude.

### `prefix_crosswalk.csv`
Filled by Step 3. Same structure with `canonical_a` / `canonical_b` completed — input for Step 4.

### `plays_with_side.csv`
`plays.csv` plus `field_pos_side`, `yardline_raw`, `yardline_100`. This is the final analysis-ready file.

## How the Crosswalk Works

PrestoSports PBP HTML uses game-specific field position prefixes like `SDMESA25`, `MSJC-FB44`, `MT. SAN31`. The same abbreviation can mean different teams across games (`MT. SAN` = Mt. San Antonio in some games, Mt. San Jacinto in others), so the mapping must be resolved per-game.

Since each game has exactly two teams, every prefix is unambiguous — it must belong to one of the two known participants. Claude resolves this by matching each prefix string against `team_1` and `team_2`.

Games with no `field_position` values in `plays.csv` (e.g. games where PrestoSports didn't record situation cells) will appear in the draft with a warning and produce no `field_pos_side` output in Step 4 — this is a data gap upstream.

---

## Canonical Model

The canonical game key is the boxscore slug, e.g. `20251018_4eui`. This is stable across team schedule pages and avoids drift from team-name suffixes in URLs.

`games.csv` is the authoritative game/team mapping. `plays.csv` is keyed to it. `plays_with_side.csv` is the analysis-ready output — use it, not `plays.csv`, for field position analysis.
